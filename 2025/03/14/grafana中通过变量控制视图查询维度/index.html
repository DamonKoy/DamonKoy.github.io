grafana 中通过变量控制视图查询维度

Posted on 2025-03-14 Symbols count in article: 1.3k Reading time ≈ 1 mins.

在 Grafana 中通过一个变量来切换时间粒度，这需要确保 SQL 查询能够与 Grafana 的变量传递机制兼容，例如通过模板变量（template variables）来动态替换时间粒度和时间范围。

总结下来，解决这个问题的步骤可能包括：

  1. 在 SQL 中根据时间粒度变量动态生成分组键。
  2. 动态调整时间范围筛选条件，以适应不同的时间粒度。
  3. 确保时间格式化和分组的正确性。
  4. 处理可能的性能优化，如索引使用。
  5. 在 Grafana 中配置模板变量以传递时间粒度参数。

现在需要将这些思考转化为具体的 SQL 实现，并验证其可行性。

关键实现说明

1. 动态时间分组逻辑

  • 10 分钟分组：
    使用 FLOOR(分钟数/10)*10 将时间对齐到最近的 10 分钟（如 09:23 → 09:20）。
  • 按天分组：
    直接取 DATE(create_time)。
  • 按周分组：
    通过 DATE_SUB(create_time, INTERVAL WEEKDAY(create_time) DAY) 将日期对齐到周一的日期（假设周起始为周一）。

2. 动态时间范围筛选

  • 按 10 分钟：查询最近 24 小时数据，确保足够的时间段覆盖。
  • 按天 / 周：分别扩展时间范围至 7 天或 4 周，避免数据稀疏。
  • Grafana 变量传递：变量 $time_granularity 的值为 10m、1d、1w，需在 Grafana 中配置下拉选项。

3. 时间戳标准化输出

  • 使用 UNIX_TIMESTAMP(time_bucket) 将时间转换为 Unix 时间戳，确保 Grafana 能正确识别为时间序列。

4. 索引优化建议

  • 若数据量大，建议为 create_time 列创建索引，并联合索引 (create_time, source, start_city_name, supplier_id)。

Grafana 配置步骤

  1. 定义时间粒度变量：
    • 进入 Dashboard 设置 → Variables → New
    • Name: time_granularity
    • Type: Query
    • Query: '10m' AS 10 分钟 , '1d' AS 按天 , '1w' AS 按周 `
    • 启用多选或单选，根据需求配置默认值。
  2. 配置 SQL 数据源：
    • 在 Panel 的 SQL 查询中，直接引用 $time_granularity 变量。
    • 确保数据库连接支持变量替换（如 MySQL 需启用预处理语句）。
  3. 时间范围联动：
    • 在 Dashboard 的时间选择器中，根据 $time_granularity 自动调整时间范围（如选择 “按周” 时，默认展示最近 4 周）。

常见问题处理

  • 时区问题：
    确保数据库和 Grafana 时区设置一致，或在 SQL 中使用 CONVERT_TZ 函数转换时区。
  • 空数据填充：
    若需补全无数据的时间段，可在 SQL 中使用日期生成表（如 generate_series）左连接业务数据。
  • 性能优化：
    按周分组时，对大数据表可能较慢，建议定期预聚合统计到周粒度表中。
